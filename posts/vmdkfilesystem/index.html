Access file system from a vmdk file

Here is a mini guide on how to access the file system from a vmdk file using 7zip

This can be done in many ways, using vmware-mount, QENU, /dev/loop and guestmount or 7zip
For this guide i used 7zip as I found it to be the easiest way.

First of all, you must check if it is possible to list the contents of the vmdk file using 7zip

  ` 7z l file.vmdk  / you can redirect the output in a text file in order to find specific files (grep)`

If the output present relevant directory (it should), that means you can extract specific files doing

  ` 7z e file.vmdk desired_file`   ***this will extract desired_file from the .vmdk***

If you need to have the entire disk simply use

  ` 7z e file.vmdk`  ***This will extract the whole archive***

The output should normaly be a .img file, you may have two or more .img file, just use the command “file” to see which one is the fylesystem. Once you did that you can then mount the .img file doing so

  ` sudo mkdir /mnt/iso`
  ` sudo mount -o loop /path/to/the/image/file /mnt/iso`

You can check everything went well by running mount , df -H or ls -l /mnt/iso

There you a file system all you can break… or use for forensic
